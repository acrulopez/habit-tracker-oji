# Habit Tracker Oji

A minimal habit tracker for iOS and Android, built with **Expo (SDK 56) + React Native + TypeScript + Expo Router**.

- Each habit is just a **name + emoji**.
- Each habit shows the **last 3 days** (−2, −1, Today) as tappable cells — tap any
  to mark it done or undo it (binary per day). Same on the widget.
- The **⋮ menu** on each habit opens **History** (a calendar of completed days +
  streak/total stats), plus Edit and Delete.
- The **history calendar is editable** — tap any past/today day to toggle it.
- **Settings** (gear icon) → **Reorder habits** with drag handles; you can also
  long-press a habit on the home list to drag it.
- A **home-screen widget** shows the last 3 days per habit and lets you complete
  any of them **without opening the app**.
- **Local-only** storage (MMKV). Light & dark mode follow the system.

## Requirements

- Node + npm
- Xcode (iOS) / Android SDK (Android) for native builds
- Widgets require a **custom dev/EAS build — they do not run in Expo Go.**
- iOS widget interactivity requires **iOS 17+**.

## Getting started

```bash
make install        # npm install
make ios            # build + run the iOS dev client (prebuilds native code)
make android        # build + run the Android dev client
make typecheck      # tsc --noEmit
make test           # jest unit tests
```

Run `make help` for all targets.

### Before building for iOS

Set your Apple Developer Team ID (needed for the App Group used by the widget):

```bash
export APPLE_TEAM_ID=XXXXXXXXXX
```

It is read by `app.config.ts` and applied to the app + widget targets.

## Architecture

```
app/                       Expo Router screens (home, new/edit modals, history)
src/
  data/                    MMKV store + HabitRepository (the cloud-sync seam)
  store/useHabitStore.ts   Zustand state, persisted via the repository
  lib/                     date keys + streak math (pure, unit-tested)
  widget/                  snapshot builder, sync bridge, reconcile, Android widget UI
  components/, theme/       UI
targets/widget/            iOS WidgetKit extension (SwiftUI + App Intent)
```

### Widget data flow

There is no write-once cross-platform widget, so each platform has its own widget
sharing one logical "today snapshot":

- **iOS:** the app writes a JSON snapshot into the **App Group `UserDefaults`** (via
  `ExtensionStorage` from `@bacons/apple-targets`) and reloads the WidgetKit
  timeline. The widget's **App Intent** toggles a habit and queues the change; the
  app drains that queue into MMKV on next foreground (`src/widget/reconcile.ts`).
- **Android:** `react-native-android-widget` renders the widget in JSX. Taps run a
  headless JS task (`src/widget/widgetTaskHandler.tsx`) that toggles MMKV directly
  and re-renders — no app launch.

`src/widget/syncWidget.tsx` is called after every mutation to keep the widget fresh.

## Notes / future work

- Storage is local-only behind `HabitRepository`; cloud sync + accounts would slot
  in there without touching the UI.
- No notifications/reminders in v1.
- Native folders (`ios/`, `android/`) are generated by prebuild and git-ignored.
